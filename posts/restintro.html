REST Architecture

-REST, representational state transfer, defines general constraints to follow while developing APIs. Provides a large amount of flexibility, as it does not demand strict implementation rules

-With REST arch., we can control data over the internet through HTTP commands

-The main idea behind REST is to provide a uniform interoperability between applications on the internet, namely in a stateless manner

-Benefits of REST arch include:

  • increased performance
  • scalability
  • extensibility
  • modifiability
  • simplicity

REST Verbs

-HTTP methods are available for interacting with resources on the server. The five main verbs used in a well-designed RESTful application are:

  • GET: read resource
  • POST: create resource
  • PUT: update existing resource (may create if none exists)
  • PATCH: partial update on resource
  • DELETE: deletes resource

—There are other HTTP verbs as well, such as HEAD, OPTIONS, etc.

-Verbs get, put, post, and delete used for CRUD operations. In practice, patch is a fifth HTTP method that’s quite useful for updating resources with partial data

Response Codes

-Communication between client and server in a RESTful system occurs primarily via HTTP response codes

-A client makes a request to the server, which is handled by the controller, mapped to an appropriate endpoint, some business logic and database interaction may occur, and a response code is sent back to the user to convey their request results

-There are five categories for response codes:

  • 1xx - Informational
  • 2xx - Success
  • 3xx - Redirection
  • 4xx - Client Error
  • 5xx - Server Error
